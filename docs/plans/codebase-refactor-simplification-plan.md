# Codebase Refactor & Simplification Plan

> **Status**: In progress — executing batches R-A → R-D (approved 2026-07-04; Phase R-E held on §5 decisions)
> **Created**: 2026-07-04
> **Goal**: A simpler, more elegant eve-horizon codebase. Decompose god files, collapse duplication, converge inconsistent patterns — in the code that *stays*.
> **Companion doc**: [platform-slim-down-plan.md](./platform-slim-down-plan.md) owns *deletion* (dead code, zero-usage features, legacy shims, docs purge). This plan owns *structure*. Where a slim-down batch removes code this plan would otherwise refactor, the slim-down batch wins and the item here is marked as gated on it.
> **Method**: read-only review (2026-07-04) of `apps/*` and `packages/*` source (`dist/` excluded), `k8s/`, `bin/eh`, and test structure. Every finding carries file:line evidence measured on branch `docs/refactor-simplification-plan` @ fork point `fcc2fa02`.
> **Verification model**: per-batch gates — `pnpm build`, `pnpm test`, `./bin/eh test integration --reset-db`, plus k3d manual scenarios (`tests/manual/scenarios/`) where runtime behavior is touched. Same gate discipline as the slim-down execution log (§6b there).
> **Citation audit (2026-07-04)**: 6 findings spot-checked against source before publication — API-5 (3 identical `resolveWorkerUrl` bodies), XSV-1 (raw namespace at `env-logs.service.ts:94` + identical `toK8sName` copies), MIG-1 (`_migrations` vs `schema_migrations` + `db-migrate-job.yaml` → `packages/db/dist/migrate.js`), SSO-1 (single 1,829-line file, `loginPageHtml:385`), SHD-1 (`claude.ts`/`mclaude.ts` diff = name strings only), all **confirmed**; CLI-2's drift **corrected** during audit (46-vs-49 keys; broken trio is `identity`/`ingest`/`providers`, not the originally-reported keys).

## 5. Decision-gated items

1. **GWY-1**: gateway plugin framework collapse assumes nostr+webchat removal (slim-down §6 decision 4). If webchat is kept, keep the registry but still drop the no-op `api` provider indirection.
2. **EH-1**: which tool owns the local k3d lifecycle — recommendation: `eve local` (shipped CLI), with `bin/eh` as thin shims. Owner should confirm the `eh`-specific conveniences to preserve (instance prefixes, `k8s_owner` guard).
3. **SDK-1**: `@eve-horizon/auth` is a *published* SDK — dropping `eveAuthMiddleware`/`eveUserAuth` is a breaking release (`sdk-v*` major bump). Pre-deployment posture says do it; confirm no external consumers beyond the sister repos.
4. **SDK-3**: only if chat SDK survives slim-down §3.3.
5. **XSV-2**: k8s client major-version bump on the worker — mechanical but subtle; decide whether to fold into R-C1 (deployer decomposition) or keep separate (recommendation: separate, before R-C1).
6. ~~MIG-1 reconciliation~~ — moot; MIG-1 withdrawn (runners serve disjoint databases).
7. **SHD-7** (ids), **API-9** (orThrow): cosmetic — include or drop at implementer's discretion.

## 6. What this plan does NOT touch

Everything the slim-down plan owns (its §3-4): dead code, zero-usage features, legacy shims, deprecated CLI commands, docs. Also explicitly out of scope here: the DB's ORM-free tagged-template SQL style (a strength), `schemas/auth.ts` composition, dashboard's data-layer/hook pattern, the gateway's `gateway-chat.service.ts` (695 LOC but cohesive), and the api integration-test bootstrap — all reviewed and judged healthy as-is.

## 7. Estimated impact

Rough, assuming slim-down G1/C-phase land first: the five god files (deployer 4,195, loop 3,516, cli job 5,002, cli help 3,238, sso main 1,829 = 17,780 LOC) decompose into focused modules with no file over ~600 lines; CLI-1/2 alone deletes ~3,000+ lines of hand-maintained metadata and fixes the three commands (`identity`, `ingest`, `providers`) whose `--help` is broken and which are undiscoverable today; dedup batches (R-B*) remove roughly 1,500-2,500 duplicated lines and, more importantly, single-source the k8s naming, migration state, auth-client, and harness-enum contracts that currently have split-brain potential. Test seams appear as a side effect (pure formatters, decomposed query factories, mockable `requestStream`), making the thin-coverage areas (cli, db) cheaply testable.
