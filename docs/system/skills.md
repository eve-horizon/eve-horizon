# Skills System

Eve Horizon now has a public split between developer skills and runtime skills.
Both still use OpenSkills `SKILL.md` content, but they are sourced and
materialized differently.

- **Developer skills** live in root `skills.txt` and are refreshed with `eve skills install`
- **Runtime skills** live in `.eve/manifest.yaml`, are pinned by `.eve/packs.lock.yaml`, and are materialized with `eve skills materialize`

## Overview

Skills use Anthropic's SKILL.md format and follow progressive disclosure:
- **Metadata** (name, description) is always available
- **Instructions** (SKILL.md body) are loaded when the skill is invoked
- **Resources** (references/, scripts/, assets/) are loaded as needed

Runtime materialization writes canonical skills to `.agents/skills/`, bridges
Claude to `.claude/skills`, and bridges Pi to `.pi/skills`. External manifest
packs are vendored into `.eve/materialized-skills/` so runtime startup remains
filesystem-only.

When listing local paths in `skills.txt`, use explicit prefixes (`./`, `../`, `/`, or `~`) to avoid ambiguity with `org/repo` identifiers.

## Directory Structure

```
your-repo/
├── skillpacks/              # Tracked skill sources (repo-local)
│   └── my-pack/
│       └── pr-review/
│           ├── SKILL.md
│           └── references/
├── skills.txt               # Manifest of skill sources
├── .agents/
│   └── skills/              # Generated by install hook (gitignored)
├── .claude/
│   └── skills -> ../.agents/skills  # Symlink when possible (gitignored)
├── AGENTS.md                # Optional repo-managed index (not auto-generated)
└── .gitignore               # Includes .agents/skills/ and .claude/skills/
```

In this repo, dev-only packs live under `private-eve-dev-skills/`.

## Workflow

### Developer Skills

Use the install path for local coding agents working on the repo:

```bash
eve skills install
```

This command reads `skills.txt` and installs each source via the upstream
`skills` CLI. It is the compatibility path for remote repos, one-off installs,
and broader developer tooling.

### Runtime Skills

Use the fast path for Eve jobs and runtime provisioning:

```bash
eve skills materialize manifest
eve skills materialize skills.txt
```

`eve skills materialize manifest` resolves runtime skills from
`.eve/manifest.yaml` and `.eve/packs.lock.yaml` without invoking `skills add`
for manifest-local or vendored manifest-remote sources.

`eve skills materialize skills.txt` is the fast deterministic path for **local**
`skills.txt` entries only. Remote `skills.txt` sources stay on `eve skills install`.

Runtime now materializes skills before `.eve/hooks/on-clone.sh` runs. The hook
no longer installs skills itself.

### Invoking Skills

Agents invoke skills via CLI:

```bash
skill read <skill-name>
```

This outputs the skill's SKILL.md content along with the base directory for resolving bundled resources.

## SKILL.md Format

```markdown
---
name: my-skill
description: Brief description of what this skill does
---

# My Skill

Instructions in imperative form (not second-person).

## When to Use

Load this skill when...

## Instructions

To accomplish X:
1. Step one
2. Step two

See references/guide.md for details.
```

### Guidelines

- **name**: Required, hyphen-case identifier
- **description**: Required, 1-2 sentence summary
- **Body**: Imperative form ("To do X, execute Y"), not "You should..."
- **Size**: Keep SKILL.md under 5,000 words; move details to references/

## Bundled Resources

| Directory | Purpose | Loaded |
|-----------|---------|--------|
| `references/` | Detailed documentation | On demand |
| `scripts/` | Executable utilities | Never (executed via CLI) |
| `assets/` | Templates, images | Never (used by scripts) |

## Search Priority

The skills CLI searches in this order (first match wins):
1. `./.agents/skills/` (project universal)
2. `~/.agents/skills/` (global universal)
3. `./.claude/skills/` (project Claude-specific)
4. `~/.claude/skills/` (global Claude-specific)

Project skills shadow global skills with the same name.

## Integration with Eve Horizon

When a job runs:
1. Worker clones the repository
2. Worker or agent-runtime materializes runtime skills before hooks
3. `.eve/hooks/on-clone.sh` runs with skills already present
4. Harness reads skills directly from `.agents/skills/`

Default runtime behavior is manifest-driven. Worker software-engineering jobs
may opt into a manifest `skill_mode` that also includes repo-local `skills.txt`
sources.

## Migration from Skill Packs

If you previously used `EVE_SKILL_PACK_ROOTS`:

1. Move skill pack contents to a repo-local pack path (e.g., `skillpacks/` or `private-eve-dev-skills/`)
2. Add each source to `skills.txt`
3. Run `eve skills install` to refresh developer installs

To migrate to AgentPacks (manifest-based packs), use:

```bash
eve migrate skills-to-packs
```

Then move the generated fragment into `.eve/manifest.yaml` under `x-eve.packs`
and remove `skills.txt` once verified.

## Examples

### Creating a Custom Skill

```bash
mkdir -p skillpacks/my-pack/my-skill
cat > skillpacks/my-pack/my-skill/SKILL.md << 'EOF'
---
name: my-skill
description: Handles X when user asks about Y
---

# My Skill

To accomplish X:
1. Check the configuration in `config/`
2. Apply the standard patterns from references/patterns.md
3. Validate the output

## References

- `references/patterns.md` - Standard patterns to apply
EOF

eve skills install
git add skillpacks/my-pack/my-skill skills.txt
git commit -m "Add my-skill"
```
