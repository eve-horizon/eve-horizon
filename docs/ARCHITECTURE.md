# Docs Architecture

> **What**: The documentation taxonomy for Eve Horizon.
> **Why**: Separates current system truth from historical plans and ideas.

## Overview

Docs are split into current truth, historical work, and ideas. Architectural decisions must land in `docs/system/`.
Use `AGENTS.md` for living project memory and link to deeper docs here.

## Key Decisions (Why)

- **System vs work split** keeps current truth clean and easy to trust.
- **Progressive disclosure** prevents long, stale mega-docs.
